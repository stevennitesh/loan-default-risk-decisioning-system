# Reports Directory

This folder contains two different classes of files:

- Curated portfolio evidence committed to Git.
- Local runtime outputs that are regenerated by the pipeline and ignored by Git.

## Curated Evidence

The committed experiment reports and selected CSV snapshots preserve the post-v1 modeling trail used by the README and Power BI comparison. These files are intentionally versioned:

- `reports/experiments/*.md`
- `reports/experiments/experiment_log.csv`
- `reports/experiments/*_selected_features.csv`
- named experiment summary CSVs such as `010_recency_stability_summary.csv`, `012_last_k_stability_summary.csv`, and `014_feature_cleanup_stability_summary.csv`
- calibration, feature-selection, and model-stability comparison CSVs referenced by experiment reports
- `reports/model_card.md`

These artifacts should change only when the corresponding experiment result is intentionally refreshed and the linked narrative is updated in the same change.

## Regenerated Runtime Outputs

Standard pipeline outputs such as `model_metrics_summary.csv`, `model_threshold_metrics.csv`, dashboard CSV bundles, figures, and DuckDB/model artifacts are generated locally and ignored by Git. Rebuild them with the Makefile targets documented in the root README.
